# Project Standard

A public release branch should meet this baseline before merge.

## Repository

- README explains the product and how to run it.
- License metadata is present.
- Contributing guide is present.
- Security policy is present.
- Code of conduct is present.
- Environment example uses placeholders only.

## Engineering

- CI runs on pull requests.
- Typecheck passes.
- Build passes.
- API smoke test passes.
- Security scan runs.
- Provider keys stay out of git.

## Product

- The app uses saved closet items only.
- Voice answers stay short.
- Maps and place context require user permission.
- Feedback affects the next outfit path.
- The web app stays simple and quiet.
